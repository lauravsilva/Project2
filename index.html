Should I Go Outside?

An app that tells you if should stay in or go out based on the weather



Features:

  • Get current weather information based on user input (city or zip code)
  • Based on the weather:
    1. Encourage the user to go outside (Between 60 and 89 degrees F) and give a randomized suggestion for an outdoor activity
    2. Give the user two options for staying in:
      1. Choose a movie for me: list a random movie based on specified genre
      2. Let me search Netflix: allow the user to search Netflix by title
  • Link to Fan.tv so the user can find where to watch the random movie
  • Link to directly watch on Netflix

Web services:

  • World Weather Online
  • The Movie Database
  • Netflix Roulette

Resources:

Besides the APIs already mentioned, these sources were also used in the making of this application:
  • Materialize
  • jQueryUI accordion
  • The Movie DB Docs
  • tmdb parser

High-Level Architecture:

This application was created with JavaScript, jQuery, HTML5 and CSS. There are three modules in the api.js file which use the respective APIs: movieDatabase, weather, netflixroulette.

Known Bug:

  • If the movie title has special characters, it will not be displayed.

Design & Technical Decisions:

My original idea for this project had to be changed as soon as I actually understood what the APIs I had chosen did. RottenTomatoes did not approve an API key for me, the NetflixRoulette did not populate a list of random choices based on a movie genre like I had hoped, and Yahoo Weather was difficult to implement. From there, I brainstormed and did more research to find alternative web services. In the end, I used two APIs that did not require an API key and one that did (Movie Database).

I kept each API separate in order to adhere to good design practices such as separation of concerns and low coupling, and each has its own method of parsing the data and displaying it. Most of the JavaScript work is done in one file, the HTML in another, and the CSS in yet another. jQuery was used to hide and show certain divs, and for the creation of the accordion. For the HTML5 technologies, I used a combination of new semantic features including a footer tag, progress bar, a datalist for suggestions, and specified the input type as search and required. I take pride in the fact that the application is easily extendable, so in the future I could implement other features such as more options for staying in or a current location button.

When it came to design, I decided to use the Materialize framework (only the CSS file). I had only ever used Bootstrap, so I wanted to try something new and create a very minimalistic yet clean UI. This was a great choice because the framework has so much to offer, from a responsive grid, to a predetermined list of colors, to little icons and much more.
Extras:
For my extra work, I used three APIs and really focused my attention on usability and creating an application that is not only aesthetically pleasing, but an overall cool / useful experience. The two column layout reduces the amount that the user has to scroll, a progress bar is used when loading, there are fade-ins and outs for different components and overall I believe there is sufficient feedback to easily understand the application. In addition, I created a randomizing algorithm so the movie chosen for the user based on genre will rarely ever repeat; the user can keep pressing the arrow button to get a new recommendation as many times as he or she may want.
